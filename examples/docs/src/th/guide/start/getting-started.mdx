# เริ่มต้นอย่างรวดเร็ว

## การเริ่มต้นโปรเจกต์

สร้างและเริ่มต้นโปรเจกต์ Gez ใหม่ โดยทำตามขั้นตอนต่อไปนี้:

```bash
mkdir your-app-name
cd your-app-name
```

## การเลือกเวอร์ชันเฟรมเวิร์ก

เฟรมเวิร์ก Gez มีเวอร์ชันสแต็กเทคโนโลยีหลายเวอร์ชัน แต่ละเวอร์ชันได้รับการปรับให้เหมาะสมกับสถานการณ์การใช้งานที่แตกต่างกัน โปรดเลือกเวอร์ชันที่เหมาะสมตามความต้องการของโปรเจกต์:

### HTML

เหมาะสำหรับโปรเจกต์ที่ต้องการโครงสร้างที่เรียบง่ายที่สุด:

- ไม่มี dependency ภายนอก ใช้งานได้ทันที
- ประสบการณ์การพัฒนา JavaScript แบบดั้งเดิม
- เหมาะสำหรับการสร้างเว็บไซต์แบบสแตติกและแอปพลิเคชันเบา
- รองรับการขยายฟังก์ชันการทำงานแบบค่อยเป็นค่อยไป

[ดูเอกสารเวอร์ชัน HTML อย่างละเอียด](../frameworks/html.md)

### Vue2

เหมาะสำหรับการพัฒนาแอปพลิเคชันระดับองค์กร:

- รองรับ TypeScript อย่างเต็มที่
- ระบบนิเวศของคอมโพเนนต์บุคคลที่สามที่หลากหลาย
- เครื่องมือการพัฒนาที่ครบวงจร
- ความเสถียรที่ผ่านการทดสอบในสภาพแวดล้อมการผลิต

[ดูเอกสารเวอร์ชัน Vue2 อย่างละเอียด](../frameworks/vue2.md)

### Vue3

เหมาะสำหรับการพัฒนาเว็บแอปพลิเคชันสมัยใหม่:

- ระบบ reactive ที่ใช้ Proxy
- รองรับ Composition API
- ประสิทธิภาพ runtime ที่ดีขึ้น
- ขนาด bundle ที่เล็กกว่า

[ดูเอกสารเวอร์ชัน Vue3 อย่างละเอียด](../frameworks/vue3.md)

### Preact+HTM

เหมาะสำหรับโปรเจกต์ที่ต้องการความเบาและประสิทธิภาพสูง:

- ขนาด runtime ที่เล็กมาก (3KB)
- ไวยากรณ์เทมเพลต JavaScript แบบดั้งเดิม
- API ที่เข้ากันได้กับ React
- ประสิทธิภาพที่ยอดเยี่ยม

[ดูเอกสารเวอร์ชัน Preact+HTM อย่างละเอียด](../frameworks/preact-htm.md)

## การพัฒนาด้วย AI ช่วยเหลือ

เฟรมเวิร์ก Gez มีความสามารถในการพัฒนาด้วย AI ช่วยเหลือ ซึ่งสามารถเพิ่มประสิทธิภาพการพัฒนาได้อย่างมาก:

1. เลือกเอกสารเวอร์ชันเฟรมเวิร์กที่เหมาะสม
2. นำเนื้อหาเอกสารไปให้ AI ช่วยเหลือ
3. AI จะสร้างโครงสร้างโปรเจกต์และไฟล์คอนฟิกโดยอัตโนมัติ

:::tip คำแนะนำ
การพัฒนาด้วย AI ช่วยเหลือไม่เพียงแต่เร่งความเร็วในการเริ่มต้นโปรเจกต์ แต่ยังช่วยให้โครงสร้างโปรเจกต์เป็นไปตามแนวปฏิบัติที่ดีที่สุด
:::
